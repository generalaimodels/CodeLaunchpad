HTML Styles: Introduction

HTML styles allow you to control the look and feel of your web page elements. Styles are most commonly written in CSS (Cascading Style Sheets), but can be applied directly in HTML using the style attribute.

1. Inline Styles

Inline styles are applied using the style attribute inside an HTML tag. They override internal and external styles for that specific element. 

<p style="color: blue;">This text is blue.</p>

This text is blue.

<h2 style="color: green; background-color: #e0ffe0; padding: 10px;">Styled Heading</h2>

Styled Heading

<span style="font-size: 24px; font-family: 'Courier New', monospace;">Large Monospace Text</span>

<div style="border: 2px solid #007acc; margin: 12px 0; padding: 8px;">Box with border</div>

<p style="color: red !important;">This is forced red text.</p>

This is forced red text.

2. Internal CSS Styles

Internal CSS is defined in a <style> block in the <head> of the HTML document. It is used to style multiple elements consistently. 

<style>
  p {
    color: #444;
    line-height: 1.6;
  }
</style>

All paragraphs are styled with internal CSS (see this file's <head>).

    background: yellow;
    font-weight: bold;

3. Common CSS Properties

  • color: Text color (color: red;)
  • background-color: Background color (background-color: #f0f0f0;)
  • font-size: Size of text (font-size: 18px;)
  • font-family: Font type (font-family: Arial, sans-serif;)
  • text-align: Text alignment (text-align: center;)
  • border: Border around elements (border: 1px solid #000;)
  • margin: Space outside element (margin: 10px;)
  • padding: Space inside element (padding: 10px;)
  • width/height: Size of element (width: 200px;)
  • display: Layout (display: block;, display: inline;)

4. Exceptions & Best Practices

  • Do not use inline styles for large projects. Inline styles make code hard to maintain and override.
  • Never use the style attribute for dynamic or reusable styling. Use classes and external CSS instead.
  • CSS property names are case-insensitive, but always use lowercase for standards.
  • Do not use deprecated HTML attributes like bgcolor, font, align. Always use CSS for styling.
  • Avoid !important in inline styles. It makes debugging and maintenance difficult. 
<!-- Deprecated: <body bgcolor="yellow"> -->
Use style="background-color: yellow;" instead.

5. Combining Inline and Internal Styles

Inline styles have the highest priority, followed by internal styles, then external styles. If multiple styles apply, the most specific one wins. 

<style>
  .priority { color: green; }
</style>
<p class="priority" style="color: red;">This will be red, not green.</p>

This will be red, not green.
