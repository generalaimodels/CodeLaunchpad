HTML Lists & Block vs Inline Elements: 10 Detailed Examples

This file demonstrates the use of HTML lists and the distinction between block and inline elements, covering syntax, attributes, accessibility, and best practices. Each example is explained in detail for clarity.

1. Basic Unordered List (<ul>)

An unordered list uses <ul> with <li> items. 

<ul>
    <li>Apple</li>
    <li>Banana</li>
    <li>Cherry</li>
</ul>
  • Apple
  • Banana
  • Cherry
  • Default marker is a disc (â€¢).
  • Use for items where order does not matter.

2. Basic Ordered List (<ol>)

An ordered list uses <ol> with <li> items. 

<ol>
    <li>First</li>
    <li>Second</li>
    <li>Third</li>
</ol>
  1. First
  2. Second
  3. Third
  • Default marker is a number.
  • Use for steps, rankings, or sequences.

3. Nested Lists

Lists can be nested for hierarchical data. 

<ol>
    <li>Fruits
        <ul>
            <li>Apple</li>
            <li>Banana</li>
        </ul>
    </li>
    <li>Vegetables</li>
</ol>
  1. Fruits
    • Apple
    • Banana
  2. Vegetables
  • Indentation and marker type change automatically.

4. Custom List Markers (CSS)

Use list-style-type in CSS to change marker style. 

<ul style="list-style-type: square;">
    <li>Alpha</li>
    <li>Beta</li>
</ul>
  • Alpha
  • Beta
  • Other values: circle, disc, decimal, lower-roman, etc.

5. Description List (<dl>, <dt>, <dd>)

Description lists are used for name-value pairs, such as terms and definitions. 

<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>
    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>
</dl>
HTML
HyperText Markup Language
CSS
Cascading Style Sheets
  • <dt>: term; <dd>: description.

6. Ordered List with start, reversed, and type Attributes

Control numbering and marker type with attributes. 

<ol start="5" reversed type="A">
    <li>Item 1</li>
    <li>Item 2</li>
</ol>
  1. Item 1
  2. Item 2
  • start: starting value.
  • reversed: counts down.
  • type: marker style (1, A, a, I, i).

7. List Accessibility (aria-label, role)

Use ARIA attributes for better accessibility, especially for custom-styled lists. 

<ul aria-label="Navigation Menu" role="list">
    <li>Home</li>
    <li>About</li>
</ul>
  • Home
  • About
  • Screen readers will announce the list as "Navigation Menu".

8. Block Elements Demo

Block elements start on a new line and take up the full width available. Examples: <div>, <p>, <h1>, <ul>, <table>. 

<div class="block-demo">This is a block element (div)</div>
<p class="block-demo">This is a block element (p)</p>
This is a block element (div)

This is a block element (p)

  • Block elements can contain other block or inline elements.

9. Inline Elements Demo

Inline elements do not start on a new line and only take up as much width as necessary. Examples: <span>, <a>, <strong>, <em>, <img>. 

This is <span class="inline-demo">inline</span> text.

This is inline text with an inline link and an image.

  • Inline elements can only contain other inline elements or text.

10. Block Inside Inline and Inline Inside Block

Exception: Block elements cannot be placed inside inline elements (invalid HTML). Inline elements can be placed inside block elements. 

<!-- Valid: Inline inside block -->
<div>This is <span>valid</span>.</div>

<!-- Invalid: Block inside inline (do not do this) -->
<span>
    <div>Invalid!</div>
</span>
This is valid (inline inside block).
  • Browsers may "fix" invalid HTML, but always follow the standard for predictable results.

Additional Notes & Exceptions

  • Use lists for semantic grouping of items, not just for indentation or layout.
  • Block elements: start on new line, can contain block/inline; Inline: flow with text, only contain inline/text.
  • Some elements (e.g., <li>, <dt>, <dd>) are only valid inside their parent list containers.
  • For custom list markers, use ::marker or list-style-image in CSS.
  • HTML5 introduced <nav>, <section>, <article> as block elements for better semantics.
