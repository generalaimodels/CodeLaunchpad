HTML Links: 10 Detailed Examples

This file demonstrates the use of HTML links in various scenarios, covering syntax, attributes, accessibility, and best practices. Each example is explained in detail for clarity.

1. Basic Link to an External Website

The simplest form of a link uses the <a> tag with an href attribute. 

<a href="https://www.example.com">Visit Example.com</a>
Visit Example.com
  • href: The destination URL.
  • By default, opens in the same tab.

2. Open Link in a New Tab

Use target="_blank" to open the link in a new browser tab. For security, always add rel="noopener noreferrer". 

<a href="https://developer.mozilla.org/" target="_blank" rel="noopener noreferrer">MDN Web Docs</a>
MDN Web Docs
  • target="_blank": Opens link in a new tab/window.
  • rel="noopener noreferrer": Prevents security vulnerabilities (tabnabbing).

3. Link to an Email Address (mailto:)

Use mailto: to create a link that opens the user's default email client. 

<a href="mailto:hello@domain.com">Email Us</a>
Email Us
  • Can prefill subject and body: mailto:hello@domain.com?subject=Hi&body=Hello

4. Link to a Phone Number (tel:)

Use tel: to create a link that initiates a phone call on supported devices. 

<a href="tel:+1234567890">Call Us</a>
Call Us
  • Works on smartphones and some desktop apps.

5. Link to a File Download

Use the download attribute to suggest downloading a file instead of navigating to it. 

<a href="sample.pdf" download>Download PDF</a>
Download PDF
  • Works for same-origin files or with proper CORS headers.
  • Can specify filename: download="MyFile.pdf"

6. Link to a Section on the Same Page (Anchor Link)

Use href="#id" to jump to an element with a matching id. 

<a href="#section10">Go to Section 10</a>
...
<h3 id="section10">Section 10</h3>
Go to Section 10

7. Link with title Attribute (Tooltip)

The title attribute provides additional information as a tooltip on hover. 

<a href="https://www.w3.org/" title="World Wide Web Consortium">W3C</a>
W3C
  • Use for supplementary info, not as a replacement for visible text (for accessibility).

8. Image as a Link

Wrap an <img> tag inside an <a> to make the image clickable. 

<a href="https://unsplash.com/" target="_blank" rel="noopener">
    <img src="https://images.unsplash.com/photo-1506744038136-46273834b3fb?w=100" alt="Nature" style="width:100px;">
</a>
  • Always provide alt text for accessibility.

9. Accessible Link with aria-label

Use aria-label to provide a descriptive label for screen readers, especially when the link text is not descriptive. 

<a href="https://twitter.com/" aria-label="Follow us on Twitter">
    <img src="twitter-icon.svg" alt="">
</a>
  • Use aria-label when the link content is an icon or image without text.

10. Disabled/Inactive Link (Best Practice)

There is no disabled attribute for <a> tags. To indicate a non-functional link, use aria-disabled="true" and prevent default action with JavaScript. 

<a href="#" aria-disabled="true" tabindex="-1" onclick="return false;" style="color:gray; pointer-events:none;">Disabled Link</a>
Disabled Link
  • Use aria-disabled="true" for accessibility.
  • Remove from tab order with tabindex="-1".
  • Prevent click with onclick="return false;" and pointer-events:none;.

Additional Notes & Exceptions

  • Always use descriptive link text for accessibility (avoid "click here").
  • Relative URLs (e.g., href="about.html") link to files in the same site.
  • Links can be styled with CSS for better UX.
  • Do not use <a> for actions (use <button> for actions, <a> for navigation).
  • Broken or missing href makes the link non-navigable and not keyboard accessible.
