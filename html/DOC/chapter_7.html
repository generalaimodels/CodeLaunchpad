HTML Images & Favicon: 10 Detailed Examples

This file demonstrates the use of HTML images and favicons in various scenarios, covering syntax, attributes, accessibility, and best practices. Each example is explained in detail for clarity.

1. Basic Image

The simplest form of an image uses the <img> tag with src and alt attributes. 

<img src="https://via.placeholder.com/150" alt="Placeholder image">
  • src: The image URL (relative or absolute).
  • alt: Alternative text for accessibility and SEO.

2. Image with Width and Height Attributes

Specify width and height to reserve space and improve layout stability. 

<img src="https://via.placeholder.com/200x100" alt="200x100 image" width="200" height="100">
  • Prevents layout shift as the image loads.
  • Use CSS for responsive sizing when needed.

3. Responsive Image with CSS

Use CSS to make images scale with the container. 

<img src="https://via.placeholder.com/400x200" alt="Responsive image" style="max-width:100%; height:auto;">
  • Ensures image fits container on all devices.

4. Image with title Attribute (Tooltip)

The title attribute shows a tooltip on hover. 

<img src="https://via.placeholder.com/120" alt="Tooltip image" title="This is a tooltip!">
  • Use for supplementary info, not as a replacement for alt text.

5. Image as a Link

Wrap an <img> inside an <a> to make it clickable. 

<a href="https://www.wikipedia.org/" target="_blank" rel="noopener">
    <img src="https://upload.wikimedia.org/wikipedia/commons/6/63/Wikipedia-logo.png" alt="Wikipedia logo" width="80">
</a>
  • Always provide alt text for accessibility.

6. Image with <figure> and <figcaption>

Use <figure> and <figcaption> to semantically group images and captions. 

<figure>
    <img src="https://via.placeholder.com/180" alt="Sample figure">
    <figcaption>Figure 1: A sample placeholder image.</figcaption>
</figure>
  • Improves accessibility and document structure.

7. Lazy Loading Images

Use loading="lazy" to defer loading images until they are near the viewport. 

<img src="https://via.placeholder.com/300" alt="Lazy loaded image" loading="lazy">
  • Improves page load performance, especially for images below the fold.

8. SVG Image Embedding

SVG images can be embedded using <img> or inline SVG for scalability and sharpness. 

<img src="https://upload.wikimedia.org/wikipedia/commons/4/4e/Pleiades_star_cluster.svg" alt="Pleiades star cluster" width="100">
  • SVGs scale perfectly at any size and are ideal for icons and logos.

9. Broken Image Handling (alt text and onerror)

If an image fails to load, the alt text is shown. You can also use onerror to provide a fallback. 

<img src="broken-link.jpg" alt="Image not found" onerror="this.src='https://via.placeholder.com/100';">
  • onerror can set a fallback image if the original fails to load.

10. Decorative Images (aria-hidden and empty alt)

For purely decorative images, use alt="" and optionally aria-hidden="true" to hide from assistive tech. 

<img src="https://via.placeholder.com/60" alt="" aria-hidden="true">
  • Screen readers will ignore this image.

Favicons: Explanation & Best Practices

  • Favicons are small icons shown in browser tabs, bookmarks, and shortcuts.
  • Use multiple <link rel="icon"> tags for different formats (ICO, PNG, SVG) for best compatibility.
  • Apple devices use rel="apple-touch-icon" for home screen icons.
  • SVG favicons are scalable but not supported in all browsers.
  • Place favicon.ico in the root directory for legacy support.

Exception: Some browsers cache favicons aggressively. Clear cache or use a new filename to update.

Additional Notes & Exceptions

  • Always provide meaningful alt text for non-decorative images.
  • Use srcset and sizes for responsive images (not shown here for brevity).
  • Do not use images for text content; use real text for accessibility and SEO.
  • Optimize images for web (compression, correct format) for performance.
  • SVGs can be inlined for advanced styling and accessibility.
